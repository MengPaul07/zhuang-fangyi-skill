# 知识路由

此目录整合原 `knowledge.md` 与 `documents.md`。使用时先按问题选择最窄文件，不要一次读完整个目录。

## 文件树

- `zhuang-fangyi.md`：庄方宜本人，身份、能力、价值观、习惯、管理员关系、孤舟。
- `wuling.md`：武陵城，城市定位、生活感、区域印象、庄方宜怎么看武陵。
- `hongshan-yan.md`：炎国、大炎、宏山科学院、天师府传统与组织气质。
- `relationships.md`：佩丽卡、弭弗、汤汤、大潘等人物关系与互动边界。
- `silt-systems.md`：息壤、天师桩、枢壤仪、侵蚀潮、裂隙、首墩。
- `trauma-duty.md`：科考站事故、重建责任、为什么不能浪费时间。

## 读取规则

- 问“庄方宜是谁 / 为什么这样 / 喜不喜欢管理员 / 孤舟是什么”：读 `zhuang-fangyi.md`。
- 问“武陵是什么 / 她怎么看武陵 / 武陵生活感”：读 `wuling.md`。
- 问“宏山 / 炎国 / 天师府 / 组织背景”：读 `hongshan-yan.md`。
- 问“息壤 / 天师桩 / 枢壤仪 / 裂隙 / 首墩”：读 `silt-systems.md`。
- 问“事故 / 前辈 / 为什么这么拼 / 为什么总一个人扛”：读 `trauma-duty.md`。
- 问人物关系时，读 `relationships.md`；问不确定设定时，优先读 `../epistemics.md`。

## 权威边界

- 本目录是整理后的知识底盘，不是原文。
- 查具体出处时读 `../raw/*.md`。
- 场景情绪与关系推进读 `../scenes/*.md`。
